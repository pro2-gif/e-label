# 전자라벨(e-label) 웹앱 개선 및 기능 추가 구현 계획서

## 🎯 목표
1. **첫 화면(메인)을 QR 생성기로 변경**: 현재 `qr_maker.html`에 있는 기능을 메인 화면(`index.html`)으로 만들고, 소비자가 스캔해서 보는 화면은 별도 페이지(`label.html` 등)로 분리합니다.
2. **전자라벨 표시 항목 추가**: 식약처 가이드라인에 맞춰 '제조번호'와 '사용기한'을 표에 추가합니다.
3. **영문 성분 번역 예외 처리(사전 기능)**: API 번역 오류를 막기 위해 '정제수 -> Water'와 같이 자주 쓰이는 성분을 지정된 영단어로 바로 번역하는 기능을 추가합니다.

## 📝 상세 변경 계획

### 1. 구조 및 화면 변경 (파일 이름 변경)
- 메인 화면 접속 시 바로 QR 코드가 보이게 하기 위해 파일들의 역할을 재배치합니다.
- `script.js` 내의 접속 URL(`E_LABEL_BASE_URL`)도 새로운 소비자용 화면 주소로 변경합니다.
- 파일명 변경: 
  - 기존 `index.html` (소비자용 뷰어) ➡️ `label.html`
  - 기존 `qr_maker.html` (QR 생성기) ➡️ `index.html` (메인)

### 2. 소비자용 전자라벨 디스플레이 개선 (`label.html`, `script.js`)
- 구글 시트 데이터 연결 순서(컬럼)에 '제조번호', '사용기한'을 새롭게 추가합니다.
- 소비자용 화면 표(Table)에 해당 두 가지 정보가 식약처 가이드라인에 맞게 표시되도록 HTML과 스크립트를 수정합니다.
- 제품명 10pt 이상 볼드 처리, 사용방법 주의사항 하이라이트 박스 등 기존에 구현된 부분을 더 확실히 다듬습니다.

### 3. 영문 성분 번역 사전 추가 (`script.js`)
- 식약처 API를 호출하기 전에, 우리가 미리 정해둔 단어장(Dictionary)을 먼저 확인하도록 로직을 추가합니다.
- **추가할 단어**:
  - 정제수 ➡️ Water
  - 펜틸렌글라이콜 ➡️ Pentylene Glycol
  - 부틸렌글라이콜 ➡️ Butylene Glycol
  - 1,2-헥산다이올 ➡️ 1,2-Hexanediol

## 🧪 검증 계획
1. 로컬 환경에서 페이지들을 열어서 메인 화면에 QR 생성기가 잘 뜨는지 확인합니다.
2. 드롭다운에서 제품을 선택했을 때 QR이 즉시 바뀌고 다운로드가 잘 되는지 테스트합니다.
3. QR이 가리키는 주소(소비자 화면)로 넘어갔을 때, '제조번호', '사용기한' 항목이 잘 나오는지 확인합니다.
4. [EN] 버튼을 눌렀을 때 '정제수'가 'Water'로 정확히 바뀌는지 번역 사전을 테스트합니다.
