# 구현 계획서: 영어(EN) 모드 시 TTS 음성 안내 버그 수정

영어(EN) 모드 상태에서 음성 안내(TTS) 버튼을 눌렀을 때, 한글 제품명 및 한글 정보가 조립되어 영어 음성 엔진이 정상적으로 발음하지 못하는 문제를 해결합니다.

## 해결 대상 문제
- 현재 `handleTts` 함수는 구글 시트 원본 데이터(한글)에서 직접 값을 읽어와 음성 텍스트를 구성하고 있습니다. 또한 제품명이 `getProductDisplayName(item, 'ko')`로 한글로 고정되어 있습니다.
- 이로 인해 영어(EN) 모드로 접속했음에도 불구하고 한글 제품명과 한글 원본 데이터가 영어 발음 엔진(`en-US`)으로 전달되어, 단어가 묵음 처리되거나 깨져서 들리는 버그가 발생하고 있습니다.

## 해결 방안
- 화면에 표시되는 테이블 정보는 이미 다국어 번역 처리가 완료된 상태로 DOM에 렌더링되어 있습니다.
- 따라서, 구글 시트의 원본 배열 데이터를 직접 가져오는 대신 **현재 화면(DOM)에 표시되고 있는 텍스트들을 그대로 긁어와 음성을 출력**하도록 수정합니다.
- 이렇게 하면 `EN` 모드일 때는 이미 번역되어 화면에 보이는 영문 제품명과 영문 스펙이, `KO` 모드일 때는 한글 제품명과 한글 스펙이 음성 엔진으로 정확하게 넘어가게 됩니다.
